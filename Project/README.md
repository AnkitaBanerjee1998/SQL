# 🍬 Candy Sales Performance Analysis using SQL

## 📌 Project Overview

This project analyzes transactional candy sales data using SQL to uncover business insights related to product performance, revenue generation, customer purchasing behavior, and division-level sales trends.

The analysis was conducted using SQLite and DBeaver, with a focus on applying real-world SQL techniques commonly used by Business Analysts and Data Analysts.

---

## 🎯 Business Objectives

The goal of this project is to answer key business questions such as:

- Which products generate the highest revenue?
- Which factories contribute the most to total sales?
- What percentage of revenue comes from each division?
- Which products outperform their division averages?
- How are products ranked based on revenue performance?
- What are the top-performing products within each division?

---

## 📂 Dataset Description

The project uses two datasets:

### orders.csv

Transaction-level sales data.

| Column | Description |
|----------|-------------|
| transaction_id | Unique transaction identifier |
| customer_id | Customer identifier |
| order_id | Order identifier |
| order_date | Date of purchase |
| product_id | Product identifier |
| units | Quantity sold |

### products.csv

Product master data.

| Column | Description |
|----------|-------------|
| product_id | Product identifier |
| product_name | Product name |
| factory | Manufacturing factory |
| division | Product division/category |
| unit_price | Selling price per unit |
